# MQ136 闻诊数据支撑说明

## 1. 本工程采用的核心方法

本地演示工程保持与借来闻诊代码一致的核心判定口径：

- 先采集 3 秒环境基线
- 再采集 5 秒呼气数据
- 对 MQ136 原始 ADC 做同类低通滤波
- 用 `final_raw - baseline_raw = delta_raw` 作为本次口气强度指标
- 判定条件采用双阈值：
  - 绝对阈值：`delta_raw >= 250`
  - 相对阈值：`delta_raw >= max(80, ceil(12% * baseline_raw))`

其中：
- `baseline_raw` = 环境阶段平均值
- `breath_average_raw` = 呼气阶段平均值
- `breath_peak_raw` = 呼气阶段峰值
- `final_raw` = `(breath_average_raw + breath_peak_raw) / 2`

这套方法的优点是与原闻诊代码保持一致，适合当前板级演示和工程比较。

## 2. 为什么不直接把 MQ136 当作医学浓度仪

MQ136 是金属氧化物气体传感器，适合做气味变化检测，但直接换算成临床级 H2S 浓度需要额外条件：

- 需要标准气体标定
- 需要固定气路和采样距离
- 需要温湿度、老化、漂移补偿
- 不同模块的负载电阻、放大和分压也会影响结果

所以本阶段更合理的表述是：

- 本系统先做“工程筛查阈值”
- 结果用于“是否存在明显口气增强趋势”的演示
- 不直接宣称为医学诊断值

## 3. 公开资料可引用的参考线

本项目演示时建议把“公开参考线”和“工程判定线”分开说。

### 3.1 工程判定线

工程判定线来自借来闻诊代码的会话逻辑：

- `delta_raw >= 250`
- 或 `delta_raw >= max(80, 12% 基线)`

这是当前系统真正执行的判定规则。

### 3.2 公开口臭参考线

公开研究和商品化口气检测资料中，常见的是以挥发性硫化物（VSC）或硫化氢（H2S）浓度作为参考：

- 便携式硫化物监测仪文献中，`80 ppb` 常被作为可疑口臭的早期参考线
- OralChroma 相关资料中，`112 ppb H2S` 常被用作口臭筛查参考点

在答辩或演示中，可以这样表述：

- `80 ppb` 和 `112 ppb H2S` 是公开资料中的口臭参考范围
- 我们当前的 RA6M5 + MQ136 系统没有做标准气体标定
- 因此系统内部仍以 `delta_raw` 为最终判定依据
- 后续可通过多人样本与标准设备对照，把 `delta_raw` 逐步映射到更稳定的经验分级

## 4. 当前演示最有说服力的数据形式

建议现场展示以下 3 组数据：

1. 串口原始日志 `SMELL_LOG`
   - 连续显示时间、raw、filtered、mv、温湿度

2. 单次会话日志 `SMELL_SESSION`
   - 显示 `baseline / breath_avg / breath_peak / final / delta / threshold / bad`

3. 离线分析脚本结果
   - 运行：

```bash
python tools\analyze_smell_log.py your_log.txt
```

输出重点：

- `baseline_avg`
- `baseline_std`
- `breath_avg`
- `breath_peak`
- `final_raw`
- `delta_raw`
- `relative_threshold_12pct`
- `noise_guard_4sigma`
- `suggested_threshold`

## 5. 演示时推荐说法

推荐统一说法：

“我们当前不是直接做医学诊断，而是先做基于 MQ136 的口气增强筛查。核心方法是先测环境基线，再测呼气阶段，用呼气相对环境的增量做判断。这个方法与原闻诊代码保持一致。与此同时，我们参考公开资料中的 80 ppb 和 112 ppb H2S 口臭参考线，作为后续建立映射关系的依据。当前系统已经能输出完整原始数据和会话结果，后续可以继续用多人样本把工程阈值校准得更稳。”
