# MQ136 闻诊实验报告（RA6M5 演示版）

## 1. 实验目的

本实验的目标不是直接给出医学诊断，而是验证以下 3 点：

1. 基于 `RA6M5 + MQ136 + SHT40` 的闻诊系统，能否稳定采集呼气/异味刺激数据。
2. 在保持借来工程核心算法不变的前提下，系统能否区分不同强度的异味响应。
3. 能否基于本板卡实测数据，建立一套可展示、可解释、可追溯的工程分级标准。

## 2. 硬件与采样链路

- MCU：`RA6M5`
- 气体传感器：`MQ136`
- 温湿度传感器：`SHT40`
- 日志输出：`UART7`, `115200 8N1`
- MQ136：`AO -> P001 (AN001)`，`5V` 供电，`10kΩ RL`
- SHT40：`SCL -> P204`，`SDA -> P414`

系统通过串口输出两类核心日志：

- `SMELL_LOG`：逐采样点输出 `raw/filtered/mv/temp/rh`
- `SMELL_SESSION`：单轮会话输出 `baseline/breath_avg/breath_peak/final/delta`

## 3. 核心算法口径

本工程保持与借来闻诊代码一致的核心方法：

- 环境基线采样：`3 s`
- 呼气/刺激采样：`5 s`
- 对 MQ136 原始 ADC 进行同类低通滤波
- `final_raw = (breath_avg + breath_peak) / 2`
- `delta_raw = max(0, final_raw - baseline_raw)`
- 原始判定线：
  - `delta_raw >= 250`
  - 或 `delta_raw >= max(80, ceil(12% * baseline_raw))`

说明：

- 本次优化没有修改核心计算路径，只在实验解释层增加了“工程分级标准”。
- 当前系统更适合表述为“口气异常筛查/异味响应分级”，不直接宣称为医学诊断。

## 4. 实验设计

### 4.1 设计思路

实验采用“空白组 - 刺激组 - 恢复组”的思路，观察系统在不同异味强度下的 `delta_raw` 变化，以及强刺激后的恢复过程。

### 4.2 推荐刺激源

本轮实验使用或规划使用的安全刺激源包括：

- 空气空白
- 正常呼气
- 生葱后呼气
- 鸡蛋后呼气
- 泡菜后呼气
- 陈醋近距离刺激

### 4.3 数据有效性判据

为了避免“没有真正吹到/没有真正刺激到传感器”造成误判，本实验增加如下有效性判据：

- 人体呼气样本：`breath_rh` 相对 `base_rh` 应有上升趋势
- 食物/挥发刺激样本：刺激动作应与 `delta_raw` 上升同步
- 若 `baseline` 长时间维持极高值，则视为“残留恢复期”，该轮不作为正式分类样本

## 7. 分阶段结果解释

由于原始日志中没有直接标注“当前刺激物名称”，因此本报告按响应阶段进行归类。后续做 PPT 时，可按实际测试顺序将阶段映射回具体食物组。

### 7.1 阶段 A：Session 1-3

特征：

- `delta = 0 ~ 2`
- 基线稳定在 `253 ~ 274`
- 系统处于低响应/空白状态

结论：

- 可作为空白组或正常背景组
- 说明系统在无明显刺激条件下噪声较小，基线可稳定复现

### 7.2 阶段 B：Session 4-6

特征：

- `delta = 74 ~ 88`
- 其中 `Session 5、6` 已触发固件 `bad=1`

结论：

- 系统已出现显著异味响应
- 说明在中强刺激下，`delta` 可以明显脱离空白组

### 7.3 阶段 C：Session 7-9

特征：

- `delta = 1 ~ 2`
- 再次回到低响应状态

结论：

- 说明刺激撤除后，系统可以恢复到接近背景水平
- 可用于支撑“不是随机漂移导致的假阳性”

### 7.4 阶段 D：Session 10-12

特征：

- `delta = 98 ~ 137`
- 三轮均为强响应
- `Session 10、11、12` 均触发 `bad=1`

结论：

- 这是当前实验中最强的响应组
- 说明系统具备对高强度挥发刺激进行显著区分的能力

### 7.5 阶段 E：Session 13-15

特征：

- `baseline = 949 ~ 1188`
- 即使 `delta` 不高，基线仍长期停留在异常高位

结论：

- 该阶段属于强刺激后的残留恢复期/传感器饱和恢复期
- 不应继续按正常轮次直接做分类结论
- 该现象反而证明系统对强刺激具有“记忆性恢复过程”，具有工程解释价值

### 7.6 阶段 F：Session 16-18

特征：

- `delta = 19 ~ 23`
- 明显高于空白组，但显著低于强刺激组

结论：

- 可定义为轻度异味响应组
- 说明系统不仅能区分“有/无响应”，还能分出中间强度层级

## 8. 最终工程分级标准

结合当前 RA6M5 板卡实测数据，建议采用如下演示版分级标准：

- `delta < 10`：无明显异味响应
- `10 <= delta < 50`：轻度异味响应
- `50 <= delta < 80`：明显异味响应
- `delta >= 80`：强异味响应
- `baseline > 800`：恢复期/残留饱和，本轮不参与正式判定

### 8.1 采用该标准的理由

- 空白组主要集中在 `0 ~ 2`
- 中等响应组集中在 `19 ~ 23`
- 明显响应已出现 `74 ~ 75`
- 强响应可达 `84 ~ 137`
- 强刺激后会出现高基线恢复区，必须单独标识

因此，这套标准能够较好反映当前硬件平台的实际响应层次。

## 9. 数据支撑结论

基于本次实测，可形成如下数据支撑结论：

1. 系统基线稳定。
   在空白/弱刺激条件下，`delta` 基本稳定在 `0 ~ 2`，说明系统噪声较低。

2. 系统对异味刺激具有分层响应能力。
   当前实测已观察到：
   - 空白组：`0 ~ 2`
   - 轻度组：`19 ~ 23`
   - 明显组：`74 ~ 75`
   - 强响应组：`84 ~ 137`

3. 原借来工程的判定公式在本平台上仍然有效，但解释层需要结合本板卡数据重新分级。
   即：核心算法不变，工程阈值解释本地化。

4. 强刺激后存在恢复期。
   `baseline > 800` 的阶段说明传感器会出现残留恢复过程，因此正式实验必须预留恢复时间。

## 10. 与公开资料的关系

公开资料中，口臭常以挥发性硫化物（VSC）或 `H2S` 浓度作为辅助参考，例如：

- `80 ppb`：常见早期可疑口臭参考线
- `112 ppb H2S`：OralChroma 相关资料常见筛查参考点

本项目当前仍不能直接把 `MQ136 ADC raw` 等同于临床浓度值，原因包括：

- 缺少标准气体标定
- 缺少固定气路
- 缺少跨个体对照样本
- 温湿度与残留恢复仍会影响读数

因此，本项目现阶段最合理的表述是：

- “建立了基于 MQ136 的口气异常筛查与异味响应分级方法”
- “后续可通过标准设备对照，把 `delta_raw` 进一步映射到更稳定的经验等级”

## 11. 当前系统的局限性

- 不能直接宣称医学诊断结论
- 对强刺激后的恢复时间较敏感
- 人体呼气样本仍需要更多带标签数据支持
- 部分轮次 `RH` 上升不明显，因此不能单靠湿度判断异味强弱

## 12. 后续优化建议

1. 在固件中加入“恢复期判定”：若 `baseline > 800`，直接提示本轮无效。
2. 在固件中加入“样本有效性提示”：若人体呼气时 `RH` 无明显变化，可提示“未形成有效呼气样本”。
3. 继续采集以下标准化数据：
   - 空白组
   - 正常呼气组
   - 葱后呼气组
   - 鸡蛋后呼气组
   - 泡菜后呼气组
   - 陈醋刺激组
4. 后续若条件允许，可与人工嗅辨或标准设备做交叉验证。
