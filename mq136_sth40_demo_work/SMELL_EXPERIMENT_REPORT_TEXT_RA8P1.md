# 基于 MQ136 与 SHT40 的闻诊系统实验报告

## 1. 课题背景与研究目的

本项目面向中医四诊中的“闻诊”场景，尝试利用气体传感器与温湿度传感器，对人体呼气或典型异味刺激进行采集与分级分析。系统硬件以瑞萨 RA 系列 MCU 为核心，前期主要在 `RA6M5` 平台完成实验验证，后续再迁移回 `RA8P1` 平台。

本实验的核心目标不是直接给出医学诊断意义上的“是否口臭”结论，而是完成以下 4 项工作：

1. 验证系统是否能够稳定采集呼气/异味刺激数据。
2. 在保持借来闻诊代码核心方法不变的前提下，验证系统是否能够对不同强度的异味刺激产生分层响应。
3. 基于本板卡实测数据，建立一套适用于当前系统的工程分级标准。
4. 论证该方法从 `RA6M5` 迁移到 `RA8P1` 时，核心算法与数据解释框架仍然成立，但绝对阈值需要进行板级标定。

因此，本项目现阶段更准确的定位应为：

- 口气异常筛查
- 异味响应分级
- 闻诊工程化验证

而不是直接替代临床口臭诊断。

## 2. 系统组成与实验平台

### 2.1 硬件组成

- 主控芯片：`RA6M5`（实验验证阶段），后续迁移目标为 `RA8P1`
- 气体传感器：`MQ136`
- 温湿度传感器：`SHT40`
- 日志串口：`UART7`

### 2.2 主要硬件连接

`MQ136` 连接信息如下：

- `AO -> P001 (AN001)`
- 模块供电：`5V`
- ADC 参考电压：`3.3V`
- 负载电阻：`10kΩ`

`SHT40` 连接信息如下：

- `SCL -> P204`
- `SDA -> P414`
- 通信方式：软件模拟 I2C

日志输出配置如下：

- 串口外设：`SCI7 / UART7`
- 波特率：`115200`
- 数据格式：`8N1`

### 2.3 日志链路

系统通过串口输出两类日志：

1. `SMELL_LOG`
   用于输出单个采样点的原始值与环境信息，包括：
   - `raw`
   - `filtered`
   - `mv`
   - `temp_c`
   - `rh`

2. `SMELL_SESSION`
   用于输出单轮闻诊会话的汇总结果，包括：
   - `baseline`
   - `breath_avg`
   - `breath_peak`
   - `final`
   - `delta`
   - `abs_th`
   - `rel_th`
   - `bad`

该日志结构既能支撑固件在线判定，也便于后续离线分析与实验报告整理。

## 3. 核心算法与方法来源

### 3.1 方法来源

本项目在优化过程中，始终保持与借来闻诊工程一致的核心逻辑，不擅自改为另一套独立的 MQ136 浓度换算算法。也就是说，本实验的基础不是直接把 `MQ136` 当成“标准浓度仪”，而是沿用原闻诊代码的“会话式增量判定方法”。

### 3.2 核心流程

每轮闻诊会话包含两个阶段：

1. 环境基线采样阶段：`3 s`
2. 呼气/刺激采样阶段：`5 s`

在采样过程中，对 `MQ136` 原始 ADC 值进行低通滤波，并计算以下特征量：

- `baseline_raw`：基线阶段平均值
- `breath_average_raw`：呼气阶段平均值
- `breath_peak_raw`：呼气阶段峰值
- `final_raw = (breath_average_raw + breath_peak_raw) / 2`
- `delta_raw = max(0, final_raw - baseline_raw)`

### 3.3 原始判定规则

借来工程原始判定规则为：

- `delta_raw >= 250`
- 或 `delta_raw >= max(80, ceil(12% * baseline_raw))`

该规则本质上是“绝对增量阈值 + 相对增量阈值”的双重判定。

### 3.4 本项目的优化原则

本次项目优化遵循以下原则：

1. 不修改核心算法流程。
2. 不改动“基线-呼气-增量”的基本思路。
3. 允许根据本板卡实测数据，对最终分级解释进行本地化。

因此，本实验的“创新点”不在于发明新算法，而在于：

- 保持原方法一致性
- 补充实验数据支撑
- 建立适配当前硬件平台的工程分级标准

## 4. 实验设计思路

### 4.1 总体思路

为了让老师能够看到系统不是“偶然跳变”，而是真正对不同异味强度有响应，本实验采用“空白组 - 刺激组 - 恢复组”的设计思路。

该思路的优点在于：

1. 能证明系统在无刺激条件下具有稳定基线。
2. 能证明系统对异味刺激具有响应能力。
3. 能证明系统响应不是永久性的，而是存在恢复过程。

### 4.2 刺激源选择

本项目优先使用安全、易获得、便于答辩解释的刺激源，包括：

- 空气空白
- 正常呼气
- 生葱后呼气
- 鸡蛋后呼气
- 泡菜后呼气
- 陈醋近距离刺激

这些刺激源的选择理由如下：

- `正常呼气`：对应真实闻诊场景
- `葱/鸡蛋/泡菜`：对应进食后口气变化
- `陈醋`：用于验证强挥发刺激响应能力

### 4.3 数据有效性判据

实验中并不是每一轮都可以直接拿来判定“口气变化”。为了避免误判，需要先区分“有效样本”和“无效样本”。

本实验采用以下有效性原则：

1. 对于人体呼气样本：
   - `breath_rh` 相对 `base_rh` 应有上升趋势
   - 若湿度几乎不变，则说明可能没有形成有效呼气采样

2. 对于直接刺激样本：
   - 刺激动作应与 `delta_raw` 上升同步
   - 若强刺激下 `delta` 仍不变化，则说明刺激没有真正进入采样区

3. 对于高基线样本：
   - 若 `baseline` 长时间维持在异常高位，说明传感器处于残留恢复期
   - 该阶段不应与正常样本混合比较

## 7. 数据分层解释

为了便于说明，本实验将 18 轮样本按响应特征分成 6 个阶段。

### 7.1 阶段 A：Session 1-3

特征：

- `delta = 0 ~ 2`
- 基线稳定在 `253 ~ 274`
- 响应极低

说明：

这一阶段可以作为空白组或正常背景组。它证明了系统在无明显刺激条件下，输出是稳定的，噪声较小，没有出现大幅随机漂移。

结论：

- 系统基线稳定
- 空白条件下 `delta` 接近 0
- 可作为后续阈值建立的参考底线

### 7.2 阶段 B：Session 4-6

特征：

- `delta = 74 ~ 88`
- 其中 `Session 5`、`Session 6` 已触发 `bad=1`

说明：

这一阶段表明系统已经对外部刺激产生明显响应。与前一阶段相比，`delta` 不再停留在 `0~2` 的背景波动水平，而是上升到接近或超过 `80` 的区间。

结论：

- 系统能够检测中强度异味刺激
- 原工程中的 `80` 级别阈值在本平台上是有实验依据的

### 7.3 阶段 C：Session 7-9

特征：

- `delta = 1 ~ 2`
- 再次回到低值区间

说明：

这一阶段的意义非常重要。它说明系统在受到前一阶段刺激后，并不会永久保持高响应，而是可以重新回到接近空白组的低响应水平。

结论：

- 系统并非随机持续漂移
- 刺激撤除后能够回落
- 有助于证明系统具备重复实验价值

### 7.4 阶段 D：Session 10-12

特征：

- `delta = 98 ~ 137`
- 三轮均属于强响应
- `Session 10 ~ 12` 全部触发 `bad=1`

说明：

这是当前实验中响应最强的一组。其 `delta` 明显高于阶段 B，也远高于空白组和轻响应组。

结论：

- 系统对高强度挥发刺激具有显著区分能力
- 强响应组与空白组、中响应组之间存在明确的层次差异

### 7.5 阶段 E：Session 13-15

特征：

- `baseline = 949 ~ 1188`
- `delta` 虽不高，但基线长时间停留在极高水平

说明：

这不是“系统失效”，而是强刺激后的典型残留恢复期。对于金属氧化物气体传感器而言，遇到高浓度刺激后出现一段恢复时间是正常工程现象。

结论：

- 高基线阶段应单独标识，不应与正常分类样本混合
- 该阶段反而说明系统确实经历了强刺激暴露
- 后续正式实验中必须预留恢复时间

### 7.6 阶段 F：Session 16-18

特征：

- `delta = 19 ~ 23`
- 高于空白组，但明显低于强响应组

说明：

这一阶段表明系统不仅能区分“有/无响应”，还能识别出介于空白与强刺激之间的中间层级。

结论：

- 系统具有分级能力，而不仅是二分类能力
- 可作为轻度异味响应区间的实验依据

## 8. 最终工程分级标准

基于当前 `RA6M5` 平台实测数据，建议建立如下工程分级标准：

- `delta < 10`：无明显异味响应
- `10 <= delta < 50`：轻度异味响应
- `50 <= delta < 80`：明显异味响应
- `delta >= 80`：强异味响应
- `baseline > 800`：恢复期/残留饱和，本轮不参与正式判定

### 8.1 该标准的依据

该标准并非凭经验随意指定，而是来自现有数据分布：

- 空白组集中在 `0 ~ 2`
- 轻度组集中在 `19 ~ 23`
- 明显组已出现 `74 ~ 75`
- 强响应组达到 `84 ~ 137`
- 强刺激后出现 `baseline > 800` 的恢复区间

因此，这套标准能够较好反映当前板卡的实际响应层次。

## 9. 数据支撑得到的核心结论

结合实验结果，可以形成以下几条最重要的结论：

### 9.1 系统具有稳定基线

空白/弱刺激条件下，`delta` 基本集中在 `0 ~ 2`，说明系统不是靠随机跳变来“碰巧有反应”，而是在低响应条件下保持较稳定的输出。

### 9.2 系统具有明显响应能力

在中强刺激条件下，`delta` 能从背景水平跃升到 `74 ~ 137`，说明系统可以有效感知异味挥发物变化。

### 9.3 系统具有分层能力

当前已经观察到以下层级：

- 空白组：`0 ~ 2`
- 轻度组：`19 ~ 23`
- 明显组：`74 ~ 75`
- 强响应组：`84 ~ 137`

这说明本系统不仅能判断“是否有反应”，还能进一步做“响应等级划分”。

### 9.4 系统存在恢复过程

强刺激后出现 `baseline > 800` 的高基线阶段，表明传感器对强刺激具有残留恢复过程。该现象符合气体传感器工程特性，也说明系统记录的不只是噪声，而是真实刺激后的动态过程。

## 10. 为什么不能直接宣称“医学口臭诊断”

这一点在答辩时必须说清楚。

尽管本系统已经能够对异味刺激做出分层响应，但当前仍不能直接把 `MQ136 ADC raw` 或 `delta_raw` 等同于临床浓度或医学诊断结果，主要原因如下：

1. 缺少标准气体标定。
2. 缺少固定气路与标准采样腔体。
3. 缺少跨个体、跨时间的大样本对照。
4. 温湿度、传感器老化、残留恢复等因素仍会影响读数。

因此，本项目当前更合理的定位是：

- 闻诊工程化验证
- 口气异常筛查
- 异味响应分级

而不是临床级口臭诊断系统。

## 11. 与公开资料的关系

公开资料中，口臭常用挥发性硫化物（VSC）或 `H2S` 浓度作为辅助参考。例如：

- `80 ppb`：常见可疑口臭早期参考线
- `112 ppb H2S`：OralChroma 相关资料中常见筛查参考点

这些公开数据的价值在于：

- 可以为项目答辩提供外部背景依据
- 可以说明“口臭检测”在公开研究中通常也依赖气体指标

但必须强调：

- 本项目当前内部执行的仍是 `delta_raw` 工程判定
- 公开 `ppb` 数据不能直接等同于本板卡 `raw` 数值

也就是说，公开参考线是“背景资料”，不是当前系统的直接执行阈值。

## 13. 当前系统的局限性

1. 不能直接输出医学诊断结论。
2. 对强刺激后的恢复时间较敏感。
3. 不同板卡、不同供电环境、不同模块批次可能导致绝对 `raw` 偏移。
4. 部分样本中湿度变化不明显，因此不能单靠 `RH` 决定异味强度。
5. 尚缺少与人工嗅辨或标准设备的系统对照。
