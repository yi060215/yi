# RA6M5 云端与手机端转接说明

## 当前主线

当前版本只围绕一条主线：

```text
终端填写手机号和检测信息
-> 后端按手机号自动建档
-> 手机端用同一个手机号登录
-> 查看历史记录和本次报告
```

手机号是患者档案和手机端账号的连接点。其他四诊信息后续统一打包到 `record` 字段里，通过 ESP32-C3 上传。

## 页面入口

入口选择页：

```text
http://127.0.0.1:5000/
```

终端信息填写与上传：

```text
http://127.0.0.1:5000/terminal
```

手机端手机号登录：

```text
http://127.0.0.1:5000/mobile
```

如果用真实手机访问，需要手机和电脑在同一个 WiFi，并使用电脑局域网 IP：

```text
http://电脑局域网IP:5000/mobile
```

## 当前验证流程

1. 打开 `/terminal`。
2. 填写手机号、性别、年龄、睡眠、疲劳、综合倾向。
3. 点击 `提交并上传`。
4. 后端立刻按手机号建档并保存报告。
5. 打开 `/mobile`。
6. 输入同一个手机号，获取本地演示验证码。
7. 输入验证码登录。
8. 在历史记录页查看刚才上传的报告。

## 核心接口

终端上传：

```http
POST /api/upload-record
```

手机端查某手机号历史记录：

```http
GET /api/patient/<phone>
```

查单次报告 JSON：

```http
GET /report/<record_id>
```

打开单次报告页面：

```http
GET /ui/report/<record_id>
```

调试用待上传记录接口仍保留：

```http
GET /api/test-record
POST /api/test-record
```

```json

  "phone": "13800138000",
  "gender": "female",
  "age": 22,
```

当前阶段先跑通手机号连接。后续 RA8P1 采集到的四诊数据统一打包进 `record`。

## 本地运行

```powershell
cd applications\aliyun_cloud
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
python app.py
```

自检：

```powershell
.\.venv\Scripts\python.exe smoke_test.py
```

自检通过时应看到：

```text
Smoke test passed
history_count=2
mobile_history_count=2
mobile_web=ok
```

## 注意事项

- 当前是本地 Flask 演示服务，不是公网云端。
- 手机访问电脑服务时不能用 `127.0.0.1`，要用电脑局域网 IP。
- GitHub 包不包含 `.venv`、数据库、JSON 运行数据、日志和编译产物。
- 真实短信还没接入，现在验证码会在页面显示，方便本地联调。
- 后续如果上线，需要部署公网服务器，并把 `BASE_URL` 和 `CLOUD_SERVER_HOST` 改成公网地址或域名。
